# Чек-лист приемки MVP

## Запуск

- [ ] Проект запускается командой `docker compose up -d --build` из `crm-backend`.
- [ ] Next/static frontend доступен на http://localhost:3002.
- [ ] Backend API доступен на http://localhost:8001.
- [ ] Healthcheck возвращает статус `ok`.
- [ ] Swagger доступен на http://localhost:8001/docs.

## Авторизация и роли

- [ ] Администратор может войти в CRM.
- [ ] Менеджер может войти в CRM.
- [ ] Без токена API заявок возвращает `401`.
- [ ] Администратор может просматривать пользователей.
- [ ] Администратор может создать пользователя.
- [ ] Администратор может заблокировать и активировать пользователя.
- [ ] Администратор может сбросить пароль пользователя.
- [ ] Менеджер не может создавать пользователей.
- [ ] Менеджер видит только свои заявки.
- [ ] Менеджер не может назначить заявку другому менеджеру.
- [ ] Админский раздел скрыт для менеджера.

## Заявки

- [ ] Пользователь может создать заявку.
- [ ] В заявке сохраняются имя клиента, контакт, курс, источник, статус, ответственный и дата следующего контакта.
- [ ] Пользователь может открыть карточку заявки.
- [ ] Пользователь может отредактировать заявку.
- [ ] Администратор может архивировать заявку.
- [ ] Архивная заявка не отображается в списке по умолчанию.
- [ ] При изменении заявки создается системный комментарий.

## Комментарии

- [ ] При создании заявки появляется системный комментарий.
- [ ] При редактировании заявки появляется системный комментарий.
- [ ] Пользователь может добавить ручной комментарий.
- [ ] Комментарии отображаются в карточке заявки.
- [ ] В UI нет действий редактирования или удаления комментариев.

## Фильтры

- [ ] Фильтр по статусу работает.
- [ ] Фильтр по курсу работает.
- [ ] Фильтр по источнику работает.
- [ ] Фильтр по ответственному менеджеру работает.
- [ ] Фильтр просроченных заявок работает.
- [ ] Поиск по имени клиента, контакту или заметке работает.

## Аналитика

- [ ] Дашборд показывает KPI: все заявки, успешные, просроченные, конверсию.
- [ ] Воронка по статусам соответствует текущей выборке.
- [ ] Графики источников и курсов соответствуют данным.
- [ ] Таблица менеджеров показывает нагрузку, просрочки и успешные заявки.
- [ ] Отображается время последнего обновления аналитики.
- [ ] Backend endpoint `GET /api/v1/reports/summary` доступен авторизованному пользователю.

## CSV

- [ ] CSV export скачивает файл с заявками.
- [ ] Экспорт учитывает права текущего пользователя.
- [ ] CSV import доступен администратору.
- [ ] Импорт показывает предпросмотр и ошибки строк.
- [ ] Валидные строки создают заявки.
- [ ] Пример `crm-frontend/public/sample-leads.csv` проходит проверку.

## Валидация API

- [ ] API не позволяет создать заявку с несуществующим статусом.
- [ ] API не позволяет назначить несуществующий курс, источник или менеджера.
- [ ] Ошибки API возвращаются понятным JSON-ответом.
- [ ] `GET /api/v1/leads/export.csv` возвращает `text/csv`.
- [ ] `POST /api/v1/leads/import.csv` возвращает отчет по созданным строкам и ошибкам.
- [ ] `PATCH /api/v1/users/{user_id}` доступен только администратору.
- [ ] `POST /api/v1/users/{user_id}/reset-password` меняет пароль пользователя.
- [ ] `PATCH /api/v1/dictionaries/statuses/{status_id}` обновляет или архивирует статус.
- [ ] `PATCH /api/v1/dictionaries/courses/{course_id}` обновляет или архивирует курс.
- [ ] `PATCH /api/v1/dictionaries/sources/{source_id}` обновляет или архивирует источник.

## Справочники

- [ ] Администратор может создать статус.
- [ ] Администратор может создать курс.
- [ ] Администратор может создать источник.
- [ ] Администратор может архивировать и активировать статус, курс или источник.
- [ ] Менеджер не видит административные действия со справочниками.
