# Рекомендации по развитию CRM

Документ фиксирует, что принято к сведению из `addition.txt`, но не должно смешиваться
с уже реализованной итерацией Next/Tailwind + аналитика + CSV + базовый RBAC.

## Уже перенесено из backlog в текущую реализацию

- Next.js + Tailwind frontend вместо статического интерфейса.
- Моковые данные для проверки всех основных сценариев.
- Дашборд руководителя: KPI, воронка, источники, курсы, менеджеры, просрочки.
- CSV export/import.
- Базовая ролевая модель `admin` и `manager`.
- Ограничение менеджера только своими заявками на уровне API.
- Поле `next_contact_date` и расчет просрочек.
- Рабочий админский экран пользователей и справочников в UI.
- Backend endpoints для блокировки/активации пользователей, сброса пароля и архивации справочников.
- Сортировка таблицы заявок по отображаемым столбцам.
- Быстрая смена статуса из таблицы заявок.

## Версия 1.1

- Inline-редактирование названий и сортировки справочников в Next UI.
- Inline-редактирование имени и email пользователя в Next UI.
- Переключение mock/API режима через отдельный env-файл frontend.
- UI для архивных заявок и фильтра `include_deleted`.
- Нормализованный импорт CSV с маппингом колонок, а не только фиксированными именами.

## Версия 1.2

- Полноценная матрица permissions вместо двух ролей.
- Роль руководителя: видит все заявки и отчеты, но не управляет системными настройками.
- Аудит изменений отдельной таблицей: старое значение, новое значение, автор, дата.
- Аудит изменения ролей и блокировок пользователей.
- Причины отказа для финальных статусов.
- Дедупликация лидов по телефону/email при создании и импорте.
- Drill-down из графиков в детальный список заявок.

## Версия 2.0

- Webhook для приема заявок с сайта.
- Интеграция с телефонией: входящий звонок, click-to-call, логирование звонков,
  ссылка на запись.
- Интеграция с почтой: входящие/исходящие письма в единой ленте коммуникаций.
- Шаблоны email с переменными.
- Экспорт отчетов в Excel и PDF.
- Настраиваемые отчеты с сохраненными фильтрами.

## Технические улучшения

- Разделить dev/prod Docker Compose.
- Добавить production reverse proxy для frontend и backend.
- Настроить HTTPS на сервере.
- Добавить резервное копирование PostgreSQL.
- Добавить CI-проверки: backend tests, frontend build, lint, Docker build.
- Добавить e2e-тесты frontend по ключевым сценариям: login, lead CRUD, CSV, dashboard.
- Перейти от wildcard CORS к списку разрешенных доменов.
- Вынести секреты из `.env` локального стенда в безопасное хранилище для prod.
